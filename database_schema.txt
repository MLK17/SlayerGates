# MCD (Model Conceptual Diagram)

Entities:
1. **User**: Represents the users of the platform.
   - Attributes: id, pseudo, email, password, avatar, created_at, role
   - Relationships: Can create tournaments, participate in tournaments, captain teams, request to join teams.

2. **Tournament**: Represents tournaments.
   - Attributes: id, title, game, description, format, max_players, is_online, location, start_date, end_date, status, created_at
   - Relationships: Created by a user, has participants, contains matches, includes teams.

3. **School**: Represents educational institutions.
   - Attributes: id, name, city, created_at
   - Relationships: Can have multiple teams.

4. **Team**: Represents teams participating in tournaments.
   - Attributes: id, name, school_id, captain_id, logo, created_at
   - Relationships: Belongs to a school, captained by a user, has members, participates in tournaments.

5. **Match**: Represents a match within a tournament.
   - Attributes: id, tournament_id, round, team1_id, team2_id, winner_id, score, status, scheduled_time, completed_time, created_at
   - Relationships: Part of a tournament, team1, team2.

6. **TournamentParticipant**: Links users to tournaments.
   - Attributes: id, tournament_id, user_id, status, seed, created_at
   - Relationships: Links a user to a tournament.

7. **TeamMember**: Links users to teams.
   - Attributes: id, team_id, user_id
   - Relationships: Links a user to a team.

8. **TournamentTeam**: Links teams to tournaments.
   - Attributes: id, tournament_id, team_id
   - Relationships: Links a team to a tournament.

9. **Leaderboard**: Represents the leaderboard of a team.
   - Attributes: id, team_id, points, wins, losses
   - Relationships: Belongs to a team.

# MLD (Model Logical Diagram)

Tables and Constraints:

1. **User**
   - Primary Key: id
   - Unique Constraints: pseudo, email
   - Foreign Key Relationships: None

2. **Tournament**
   - Primary Key: id
   - Foreign Key: creator_id references User(id)

3. **School**
   - Primary Key: id
   - Unique Constraints: name

4. **Team**
   - Primary Key: id
   - Unique Constraints: (name, school_id)
   - Foreign Key: school_id references School(id), captain_id references User(id)
   - Check Constraints: logo format (JPG, JPEG, PNG, WEBP)

5. **Match**
   - Primary Key: id
   - Foreign Key: tournament_id references Tournament(id), team1_id references Team(id), team2_id references Team(id)

6. **TournamentParticipant**
   - Primary Key: id
   - Unique Constraints: (tournament_id, user_id)
   - Foreign Key: tournament_id references Tournament(id), user_id references User(id)

7. **TeamMember**
   - Primary Key: id
   - Foreign Key: team_id references Team(id), user_id references User(id)
   - Unique Constraints: (team_id, user_id)

8. **TournamentTeam**
   - Primary Key: id
   - Foreign Key: tournament_id references Tournament(id), team_id references Team(id)

9. **Leaderboard**
   - Primary Key: id
   - Unique Constraints: team_id
   - Foreign Key: team_id references Team(id)

# Table Completion Responsibility

1. **User**
   - Completed by: Users
   - Description: Users sign up and manage their own profiles.

2. **Tournament**
   - Completed by: Admins
   - Description: Admins create and manage tournaments.

3. **School**
   - Completed by: Admins
   - Description: Admins add and manage school information.

4. **Team**
   - Completed by: Users
   - Description: Users create and manage their own teams.

5. **Match**
   - Completed by: Admins
   - Description: Admins set up matches within tournaments.

6. **TournamentParticipant**
   - Completed by: Users
   - Description: Users register to participate in tournaments.

7. **TeamMember**
   - Completed by: Users
   - Description: Users join teams.

8. **TournamentTeam**
   - Completed by: Admins
   - Description: Admins link teams to tournaments.

9. **Leaderboard**
   - Completed by: System
   - Description: The system updates the leaderboard based on match results.
